========================================
(1) ITEM
========================================

For the ITEM JSON you will generate well-structured JSON for a single ITEM using the template and rules below. This will be imported into FoundryVTT and must conform to the D&D 5e system. You must:

- Output valid JSON
- Escape all HTML double quotes (") in description strings
- Avoid trailing commas or duplicate keys
- Use null or empty string for optional empty fields
- Set all mandatory fields
- Include optional module flags only when a partial section below requires them
- Use only straight ASCII apostrophes ('). Do not use curly or typographic apostrophes (’). Do not use smart quotes. Never output typographic or curved quotes (" " " ").

The importer also accepts a native Foundry Item export containing top-level `name`, `type`, and `system` fields. Native imports preserve the complete dnd5e Item data, including activities, effects, flags, and type-specific system fields, while discarding the original document ID, folder, ownership, and source statistics so a new Item can be created safely. The generated template below remains Blacksmith's simpler flat format.

----------------------------
JSON CORE FIELD RULES & EXPECTATIONS
----------------------------

Replace the ALL-CAPS placeholders below using the rules that follow. All core template fields must be present in the JSON object. See USES AND ACTIVITIES BY ITEM TYPE for how to set uses/activities on non-consumables.


ITEMNAME: (STRING) (REQUIRED) The full item name (max 50 characters)

ITEMDESCRIPTION: (STRING) (REQUIRED) An HTML-flavored narrative describing the item's look, purpose, and history. Use paragraph, bulleted lists, bold text, and heading tags as needed.


ITEMDESCRIPTIONUNIDENTIFIED: (STRING) (REQUIRED)  An HTML-flavored narrative describing the item with vague details.

ITEMDESCRIPTIONCHAT: (STRING) (REQUIRED) A simple sentence describing the item that can be sent to the chat.

ITEMGMNOTES: (STRING) (OPTIONAL) Private GM-only notes about this item — secrets, plot hooks, intended use, ties to NPCs or quests, or "reveal after..." guidance. Never shown to players; stored privately for the GM. Use HTML (paragraphs, bulleted lists, bold) like the description. Do NOT restate the public description. Omit the field or set it to "" when there is nothing worth noting.

ITEMTYPE: (STRING) (REQUIRED) Item type. The D&D 5e item document type only. One of: 
-"Consumable", 
-"Container", 
-"Equipment", 
-"Loot", 
-"Tool", 
-"Weapon",
-"Feature",
-"Spell"

Feature and Spell are non-physical Item types. Their appended import profile defines the authoritative type-specific fields; use 0/false/empty values for irrelevant physical fields in the core object.

ITEMSUBTYPE: (STRING) (REQUIRED)
- Consumable: "Ammunition", "Food", "Poison", "Potion", "Rod", "Scroll", "Trinket", "Wand", "Wondrous Item".
- Container: null
- Equipment: "Clothing", "Ring", "Rod", "Trinket", "Vehicle Equipment", "Wand", "Wondrous Item", "Light Armor", "Medium Armor", "Heavy Armor", "Natural Armor", "Shield"
- Loot: "Art Object", "Adventuring Gear", "Gemstone", "Junk", "Material", "Resource", "Trade Good", "Treasure"
- Tool: "Artisan's Tools", "Gaming Set", "Musical Instrument"
- Weapon: "Simple Melee", "Simple Ranged", "Martial Melee", "Martial Ranged", "Natural", "Improvised", "Siege Weapon"

ITEMSUBTYPENUANCE: (STRING) (OPTIONAL) The delivery mechanism for the item:
- If ITEMSUBTYPE is "Poison" set to one of: "contact", "ingested", "inhaled", "injury"
- Use "" for all other ITEMSUBTYPE

ITEMRARITY: (STRING) (REQUIRED) Always set to one of:
- "common"
- "uncommon"
- "rare"
- "very rare"
- "legendary"

ITEMQUANTITY: (INTEGER) (REQUIRED) Set to the appropriate number of item. Default 1.

ITEMWEIGHT: (INTEGER) (REQUIRED) Always set to a number between 0 and 5 (decimals OK). Never leave blank.

ITEMPRICE: (STRING) (REQUIRED) Always set to a SP or GP value as a string (e.g., "50 GP"), or "0 GP" if truly priceless. Never leave blank.

ITEMIDENTIFIED: (BOOLEAN) (REQUIRED) the value should be true or false (not a string). Default is true.

ITEMISMAGICAL: (BOOLEAN) (OPTIONAL) Identifies if the item has magical properties. The value should be true or false (not a string).

MAGICALATTUNEMENTREQUIRED: (STRING) (OPTIONAL) If magical, set to one of the following: attunement not required,  attunement required,  attunement optional

ITEMIMAGEPATH: (STRING) (OPTIONAL) Set to ""

ITEMIMAGENUANCE: (STRING) (OPTIONAL) Defined special rules for the image. Optional.

ITEMIMAGETERMS: (ARRAY) (OPTIONAL) This is an array of synonyms for the item image that might help auto-match the image with FoundryVTT's in-built images. These strings are separated by commas. e.g. for a potion you'd include the main term and related terms: "potion", "vial", "elixir", "phial"

LIMITEDUSES: (INTEGER) (REQUIRED) Number of uses before the item is consumed. Set to 1 for single-use items, or higher for multi-use items.

LIMITEDUSESSPENT: (INTEGER) (OPTIONAL)  How many times the item has been used. Defaults to 0 unless the item was already used.

LIMITEDUSESMAX: (INTEGER) (REQUIRED)  Determines how many times something can be used. Defaults to 1.

DESTROYONEMPTY: (BOOLEAN) (REQUIRED)  Determines if the item is destroyed after use. When no more uses, the item is destroyed. e.g. an Apple would be destroyed after use where a trinket might be able to be reused. Default is true.

RECOVERYPERIOD: (STRING) (OPTIONAL) Determines how often the item resets its uses. For instance the magic may recharge every short rest and the uses get reset to their defaults. Should be set to one of the following: none, Long Rest, Short Rest, Day, Dawn, Dusk, Initiative, Start of Turn, End of Turn, Each Turn, Recharge

ACTIVITIES: (ARRAY) An array of activity objects. Each activity object should have:
- activityType: (STRING) One of: Attack, Damage, Heal, Save, Utility. Other modern dnd5e activity schemas are not yet supported by the friendly importer.
- activityName: (STRING) Display name for the activity (e.g., "Heal", "Attack", "Cast Spell")
- activityIcon: (STRING) Icon path for the activity (optional, can be left empty for default)
- activityFlavorText: (STRING) Text displayed in chat when the activity happens
- activationType: (STRING) action, bonus, reaction, passive, special, etc.
- activationValue: (NUMBER OR NULL) Usually 1 for action/bonus/reaction; null for passive/special.
- damageFormula: (STRING) Damage dice/formula, e.g. "3d8" or "2d4+2".
- damageType: (STRING) dnd5e damage type key, e.g. "fire", "slashing", "radiant".
- healingFormula: (STRING) Healing dice/formula.
- healingType: (STRING) "healing" or "temphp".
- saveAbility: (STRING) Ability key for Save activities, e.g. "dex".
- saveDC: (NUMBER OR STRING OR NULL) Fixed/formula DC; null uses normal actor/spell calculation.
- onSave: (STRING) "none", "half", or "full".
- attackType: (STRING) "melee" or "ranged".
- attackAbility: (STRING) Ability key, "spellcasting", or "" for automatic.
- rollFormula: (STRING) Optional generic roll for Utility activities.
- activityUsesMax: (NUMBER, STRING, OR NULL) Optional activity-specific uses.
- activityRecoveryPeriod: (STRING) Optional recovery period using the same labels as the profile, including "Recharge".
- activityRecoveryFormula: (STRING) Required when activityRecoveryPeriod is "Recharge"; use "6" for Recharge 6 or "5" for Recharge 5–6.
- activityDuration: (OBJECT) Activity-specific duration: {"value": 1, "units": "minute", "special": "", "concentration": false}. Use units "inst" for instantaneous.
- activityRange: (OBJECT) Activity-specific range: {"value": null, "units": "self", "special": ""}, or a measured range such as {"value": 60, "units": "ft", "special": ""}.
- activityTarget: (OBJECT) Targeting and optional measured area. Fields: affectsType (self, ally, enemy, creature, object, creatureOrObject, any, willing), affectsCount, choice, special, templateType (circle, cone, cube, cylinder, line, radius, sphere, square, wall, or ""), templateSize, templateWidth, templateHeight, templateCount, contiguous, units, and prompt. choice, contiguous, and prompt are JSON BOOLEAN values: use true or false, never "", "true", or "false". Match the template shape to the complete mechanic, including facing or direction: use circle for an omnidirectional area around the source and cone for a directional gaze, breath, or similar projected area. For either 15-foot self-originating area, use activityRange self plus affectsType creature, templateSize 15, units ft, and prompt true.
- appliedEffects: (ARRAY) Optional effects linked to this activity. Each effect supports name, img, description, statuses (dnd5e condition IDs such as "charmed", "grappled", or "restrained"), durationSeconds, durationRounds, durationTurns, changes (Foundry Active Effect changes), flags, and onSave. For a failed-save condition set onSave false. Rules that cannot be represented by a status/change—such as ending when the target takes damage—must remain in the description for GM adjudication.

USES AND ACTIVITIES BY ITEM TYPE:
- Consumable items with a real use effect (potion, poison, scroll, etc.): set itemLimitedUses / limitedUsesMax appropriately (often 1), destroyOnEmpty true when consumed, and include activities describing the effect.
- Loot, Equipment, Tool, Weapon, and Container items that are NOT actively used (no drink/heal/attack effect): set itemLimitedUses and limitedUsesMax to 1, limitedUsesSpent to 0, destroyOnEmpty to false, itemRecoveryPeriod to "none", and activities to [].
- If an item is both a crafting material and usable (e.g. a medicinal herb), use itemType "Consumable" with the correct itemSubType and full uses/activities—not Loot with fake activities.

ITEMSOURCE: (STRING) The sourcebook or hombrew source name. Default to the campaign name, [ADD-CAMPAIGN-NAME-HERE], if provided.

ITEMLICENSE: (STRING) License or attribution for the item (e.g., "OGL", "CC BY 4.0", or a custom attribution string). Default to "CC BY 4.0".
