--------------------------------
JSON FIELD RULES
--------------------------------

- "journaltype": Always "location"
- "foldername": Journal folder for this entry
- "journalname": Journal entry name (container)
- "title": The location name (city/town/settlement/site)
- "locationimage": Relative Foundry path or "None"

Location card fields:
- "cardimagetitle": A specific title for the image (e.g., "Daggerfall Docks")
- "carddescriptionprimary": 2-3 sentences about the location; this appears above the image
- "carddescriptionsecondary": 3-5 facts about the location; use an HTML list (`<ul><li>...</li></ul>`) and place it below the image

Card behavior rules:
- Card title is always the same as "title" (location name)
- Card image is always the same as "locationimage"

Section fields (use HTML or plain text):
- "introduction": About the city/location
- "geography": Environment, flora, fauna, terrain, climate
- "government": Politics, law and order, defense
- "trade": Goods, services, routes, nearby major cities and distance context
- "culture": Daily life, customs, festivals, social structure
- "religion": Faiths present and role in civic life
- "history": Founding, growth, turning points, decline/prosperity
- "notablelocations": Key places such as inns, guilds, temples, markets, keeps, etc.

If a field has no content, use an empty string.
Defaults used by importer when omitted:
- foldername -> "Libraries"
- journalname -> "Locations"
- locationimage -> "modules/burden-of-knowledge/assets/images/illustrations/common-general/common-landscape-forest.webp"

FRAMING RULES

- Avoid panoramic emptiness

STYLE RULES (ALWAYS ENFORCED)

Sketch-like outlines and textured brush strokes.

No smooth airbrushing.
No photorealism.
No hyper-polished cinematic realism.

Cinematic contrast but painterly execution.

Art direction:
Fantasy concept art for a tabletop RPG campaign.
Dramatic but grounded.
Hand-illustrated look.
High detail.
Sharp focus.
Single cohesive image.
